# Codex 任务执行引导

## continue 指令处理流程

当接收到 "continue to next task" 指令时：

1. **读取 TODO.md**：检查当前目录下的 TODO.md 文件
2. **找到下一个任务**：定位第一个未标记完成的任务（[ ] 标记）
3. **执行任务**：按照任务描述执行相应操作
4. **更新状态**：完成后将 [ ] 改为 [✓]
5. **错误处理**：遇到错误时输出详细信息并停止

## 任务执行要求

- 每次只执行一个任务
- 输出清晰的执行日志
- 任务完成后必须更新 TODO.md

## 运行约束

- 允许在必要时启动 GUI 界面进行功能验证，包括启动器和本地可视化界面
- 允许在必要时运行 API 服务或前端开发服务器进行联调和排错
- 默认仍优先采用命令行、静态分析、文件读取、文件修改等低开销方式
- 只有在验证功能、定位问题或确认修复效果时才启动 GUI、API 服务或开发服务器
- 避免启动与当前任务无关的窗口、服务或长时间驻留进程
